How to read this report

How DAT works

  1. Detect the project ecosystem(s) and select the applicable scanners.
  2. Preflight: verify the app has the inputs needed for a meaningful scan (POC→Enterprise readiness).
  3. Run security/quality/test scanners AND per-component evaluators (fail-safe, robustness, coherence).
  4. Deduplicate findings, compute the readiness score, and evaluate the Quality Gate.
  5. Emit human reports (console / HTML / PDF) and a machine fix-manifest for coding agents (Claude Code).

The Quality Gate

The Quality Gate is the deploy decision. It FAILS when the scan finds any issue at a severity listed in `failOn` (default: CRITICAL, HIGH), or when a scanner errored. Otherwise it PASSES. This is independent of the readiness score.  This run blocks on: CRITICAL, HIGH.

What the severities mean

CRITICALAn exploitable flaw or breaking defect that puts production at immediate risk.Must be fixed before deploying.
HIGHA serious security or correctness issue likely to cause harm in production.Fix before release; blocks the gate by default.
MEDIUMA real weakness that increases risk but is not immediately exploitable.Plan a fix soon; advisory by default.
LOWA minor or cosmetic issue with limited impact.Fix opportunistically.
INFOInformational context, not a defect.No action required.

What the finding categories mean

SecurityA vulnerability an attacker could exploit.Directly exposes the application or its data to attack.
DefectThe application behaves incorrectly (e.g. a failing test).Broken logic reaches users and erodes trust.
RobustnessMissing resilience: no timeout, no error handling, no input validation.The app fails unpredictably under load, bad input, or upstream outages.
CoherenceComponents disagree (e.g. a UI call that an endpoint will reject).Mismatches cause runtime failures that unit tests miss.
Fail-safeA control lacks a safe default or handled failure path.When something goes wrong, the app does the unsafe thing.
Best practiceDeviation from a hardening or quality convention.Accumulated drift makes the system harder to secure and maintain.
Coverage gapA check could not run (missing tool or input), so this area is unverified.An unverified area is not the same as a safe one.

The readiness score

Range 0–100 (higher is healthier). Bands: 80–100 (green) — Healthy — minor issues only.; 50–79 (yellow) — Caution — address before the next deploy.; 0–49 (red) — Blocked — critical/high findings present.; Severity dominates volume (one CRITICAL outweighs many LOWs) and repeated findings of the same severity have diminishing impact. The score is a health indicator — the pass/fail decision is the Quality Gate, not the score.
